# Crown Telegram bot

Уведомления контент-мейкеру в телегу: донаты, дедлайны мини-игр, выплаты, месячная
статистика. Плюс режим основателей — статистика по всей платформе.

## Запуск (2 минуты)

1. В Telegram открой **@BotFather** → `/newbot` → придумай имя → скопируй токен.
2. Запусти сайт как обычно: `npm run dev` (localhost:3000).
3. В соседнем терминале:

```bash
TELEGRAM_BOT_TOKEN=123456:ABC-твой-токен npm run bot
```

Всё. Дальше в кабинете: **Settings → Telegram → Connect** — кнопка откроет чат с ботом,
жмёшь Start, готово.

## Что умеет

| Команда | Что делает |
|---|---|
| `/settings` | вкл/выкл категории уведомлений кнопками прямо в чате |
| `/stats` | сколько сейчас (сегодня, неделя) |
| `/monthly` | итог месяца: заработано, VIP'ы (+сколько за месяц), лучший день |
| `/demo` | три образца уведомлений — посмотреть, как выглядят |
| `/stop` | отключить |

Категории (те же, что в колокольчике на сайте): ⚡ требует решения · 💸 деньги ·
✨ приятное · 📊 сводки · ⚠️ проблемы. Отключается любая, месячный дайджест — отдельно.

## Режим основателей

В чате с ботом: `/founder crown-founder` (секрет меняется через `FOUNDER_SECRET=...`).
После этого `/platform` — количество КМ, донаты за месяц, сыгранные мини-игры.

## Как устроено

`bot.mjs` — тупая труба без зависимостей: пересылает апдейты Telegram на
`localhost:3000/api/telegram/event` и рассылает то, что сайт положил в
`/api/telegram/outbox`. Вся логика — на стороне сайта
(`lib/server/telegram-store.ts`), хранилище — `bot/data/store.json` (в гитигноре).

Когда появится настоящий бэкенд, он просто зовёт `POST /api/telegram/notify`
`{handle, kind, title, body}` — бот и панель в кабинете не меняются.
